# Glossary

- `canonical`: the source definition that other artifacts derive from
- `runtime adapter`: translation layer from canonical model to a specific tool runtime
- `handoff`: structured transfer of work and context between agent roles
- `pipeline`: ordered workflow composed of agent roles and validation gates
- `profile`: project-specific configuration that specializes the canonical model
- `skill`: reusable capability package for a bounded workflow or domain
- `tool`: deterministic executable used by the agent layer
- `artifact`: any file or output produced by an agent, tool or runtime sync process

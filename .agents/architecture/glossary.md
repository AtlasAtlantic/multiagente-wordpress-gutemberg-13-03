# Glosario

- `canonical`: definición fuente de la que derivan otros artefactos
- `runtime adapter`: capa de traducción entre el modelo canónico y un runtime específico de herramienta
- `handoff`: transferencia estructurada de trabajo y contexto entre roles de agente
- `pipeline`: flujo de trabajo ordenado compuesto por roles de agente y puertas de validación
- `profile`: configuración específica de proyecto que especializa el modelo canónico
- `skill`: paquete de capacidad reusable para un flujo o dominio acotado
- `tool`: ejecutable determinista usado por la capa de agentes
- `artifact`: cualquier archivo u output producido por un agente, una tool o un proceso de sincronización runtime
